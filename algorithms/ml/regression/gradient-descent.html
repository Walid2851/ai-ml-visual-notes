Theory

Gradient Descent

An optimization algorithm that minimizes a loss function by iteratively moving in the direction of steepest descent.

Imagine you are blindfolded on a hilly terrain and want to reach the lowest valley. Each step, you feel the slope under your feet and step in the downhill direction. The learning rate controls how large each step is — too large and you overshoot the valley, too small and it takes forever.
ConceptExplanation
Loss functionMeasures how wrong the model is. We want to minimize it.
GradientThe slope of the loss — tells us which direction is "uphill".
Learning rate αStep size. Small = slow but stable. Large = fast but may diverge.
ConvergenceWhen the gradient is near zero — we've reached a minimum.
Local minimumA valley that isn't the lowest point globally (not an issue for convex loss).
Algorithm
// Minimize loss function L(w) by updating weight w

GradientDescent(α, max_steps):
  w ← 0                          // start at a random weight

  for step = 1 to max_steps:
    loss     ← L(w)             // compute current loss
    gradient ← dL_dw(w)         // compute slope at w

    w ← w - α × gradient        // step opposite to gradient

    if |gradient| < ε:           // converged — stop early
      return w

  return w                       // return best weight found
