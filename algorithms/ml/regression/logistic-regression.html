Theory

Logistic Regression

A binary classification algorithm that predicts the probability that a point belongs to class 1.

Logistic regression is linear regression with a sigmoid function wrapped around the output. Instead of predicting a number, it predicts a probability between 0 and 1. The decision boundary is the line where the predicted probability equals 0.5 — anything above goes to class 1, anything below to class 0.
ConceptExplanation
Sigmoid σ(z) σ(z) = 1 / (1 + e⁻ᶻ) — squashes any value into (0, 1). This is the predicted probability.
Log-loss The loss function: penalises confident wrong predictions heavily. −[y·log(ŷ) + (1−y)·log(1−ŷ)]
Decision boundary The line where w·x + b = 0, which gives σ = 0.5. Points on one side → class 1, other side → class 0.
Training Gradient descent on log-loss — same update rule as linear regression but with the sigmoid applied first.
vs Linear Regression Linear regression predicts a continuous value. Logistic regression predicts a class probability.
Algorithm
// Binary classification: predict P(y=1 | x) using sigmoid

sigmoid(z):
  return 1 / (1 + exp(−z))

LogisticRegression(data, α, epochs):
  w1 ← 0,  w2 ← 0,  b ← 0     // 2 features + bias

  for epoch = 1 to epochs:
    z    ← w1·x1 + w2·x2 + b       // linear combination
    ŷ   ← sigmoid(z)              // predicted probability
    err  ← ŷ − y                    // error (same as linear!)

    loss ← −mean(y·log(ŷ) + (1−y)·log(1−ŷ))   // log-loss

    dw1 ← mean(err · x1)            // gradients
    dw2 ← mean(err · x2)
    db  ← mean(err)

    w1 ← w1 − α · dw1              // update weights
    w2 ← w2 − α · dw2
    b  ← b  − α · db

  return w1, w2, b
